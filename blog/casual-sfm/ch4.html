Reprojection Error

The standard metric for SfM accuracy. Each 3D point X was originally observed at pixel p. We re-project X through the estimated camera and measure how far the projection lands from p. If the geometry is correct, the projection should land almost exactly on p.

Math

For each point Xᵢ and observation pᵢ:

  p̂ᵢ = project(Xᵢ)   (via K, R, t)
     = K (R Xᵢ + t) / (R Xᵢ + t)[2]

Reprojection error:
  eᵢ = ‖p̂ᵢ − pᵢ‖₂   (Euclidean, pixels)

Summary statistics:
  mean   = (1/N) Σ eᵢ
  median = sorted eᵢ at 50th percentile
  p95    = sorted eᵢ at 95th percentile

Code: evaluate()

cam0, cam1 = reconstruction.cameras
pts0, pts1 = reconstruction.pts2d

# Re-project 3D points into each camera
proj0 = cam0.project(reconstruction.points)
proj1 = cam1.project(reconstruction.points)

# Euclidean distance per point
err0 = np.linalg.norm(proj0 - pts0, axis=1)
err1 = np.linalg.norm(proj1 - pts1, axis=1)

print(f"cam1  mean: {err0.mean():.2f} px")
print(f"cam1  med:  {np.median(err0):.2f} px")
print(f"cam1  p95:  {np.percentile(err0,95):.2f} px")
What sub-pixel error means 0.50 px mean error on a ~3000×2000 px image is excellent for a from-scratch implementation. It means the estimated F, R, t, and 3D positions are internally self-consistent, not that the 3D positions are metrically accurate (scale is always unknown in monocular SfM).

Back-Projection Visualisation

The back-projection plot overlays each detected keypoint (circle) with its re-projected location (cross), coloured by error magnitude. Purple → near-zero; yellow → ~1 px or more. 

def plot_backprojection(cam0, cam1, pts0, pts1, proj0, proj1, err0, err1, out_path):
    fig, axes = plt.subplots(1, 2, figsize=(20, 9))
    max_err   = max(np.percentile(err0, 95), np.percentile(err1, 95), 1.0)
    cmap      = plt.cm.plasma

    for ax, cam, pts, proj, err in zip(
        axes, [cam0,cam1], [pts0,pts1], [proj0,proj1], [err0,err1]
    ):
        img = cv.cvtColor(cv.imread(cam.image_path), cv.COLOR_BGR2RGB)
        ax.imshow(img)
        c = cmap(np.clip(err / max_err, 0, 1))
        ax.scatter(pts[:,0], pts[:,1],   s=8, facecolors='none',
                   edgecolors=c, linewidths=0.6)       # detected (circles)
        ax.scatter(proj[:,0], proj[:,1], s=6, marker='+',
                   c=err, cmap='plasma', vmin=0, vmax=max_err)  # projected (crosses)

    sm = plt.cm.ScalarMappable(cmap=cmap, norm=plt.Normalize(0, max_err))
    fig.colorbar(sm, ax=axes).set_label('Reprojection error (px)')

The fountain's carved stone and brick wall produce dense, accurate matches across the full image extent. Most points show purple, meaning sub-pixel error. The few yellow points are mostly at the top-left sky region where texture is sparse.

Error Distribution: eval_errors.png

Histograms of per-point reprojection error for each camera. Both cameras show a similar distribution: bulk of points below 0.6 px, long flat tail out to ~1.2 px. 

def plot_error_histogram(err0, err1, out_path):
    fig, axes = plt.subplots(1, 2, figsize=(12, 4), sharey=True)
    for ax, err, label, color in zip(
        axes, [err0, err1], ['Camera 1', 'Camera 2'], ['#4C72B0', '#DD8452']
    ):
        bins = np.linspace(0, min(err.max(), 20), 60)
        ax.hist(err, bins=bins, color=color, edgecolor='white', linewidth=0.3)
        ax.axvline(err.mean(),      color='red',    linestyle='--',
                   label=f'mean {err.mean():.2f}px')
        ax.axvline(np.median(err),  color='orange', linestyle='--',
                   label=f'median {np.median(err):.2f}px')
Observed numbers Camera 1: mean 0.50 px, median 0.47 px. Camera 2: mean 0.53 px, median 0.50 px. The flat tail (points between 0.6 px and 1.2 px) comes from far-away wall texture where the small baseline between the two shots gives poor triangulation geometry.

Depth Distribution: eval_depth.png

Histogram of the z-coordinate of all 701 triangulated points, their distance from camera 1 along the optical axis, in scene units.

Math

depth of point Xᵢ (in camera 1 frame):
  z = Xᵢ[2]   (world = camera 1 frame)

Expected structure for a fountain scene:
  - Fountain body: foreground, smaller z
  - Brick wall:    background, larger z
  - Sky/frame:     clipped (far away)

Code

def plot_depth_histogram(pts3d, out_path):
    depths = pts3d[:, 2]
    p1, p99 = np.percentile(depths, [1, 99])
    bins = np.linspace(max(0, p1), p99, 60)
    ax.hist(depths, bins=bins, color='#55A868')
    ax.axvline(np.median(depths), color='red',
        label=f'median {np.median(depths):.1f}')

The bimodal shape is clear in the plot: a sharp peak near 2.7–2.9 (the fountain body) and a secondary cluster near 3.0–3.5 (the brickwork behind it). The long right tail corresponds to the far brick wall.

3D Visualisation with Rerun

Rerun is an open-source time-series visualiser that handles 3D primitives natively. rr.Points3D renders the point cloud; rr.Transform3D and rr.Pinhole render the camera frustums.

What gets logged

world/points
  → N coloured 3D points (the point cloud)

world/camera/0
  → Transform3D: R=I, t=0 (cam 1 at origin)
  → Pinhole: K[0,0], K[1,1], K[0,2], K[1,2]

world/camera/1
  → Transform3D: R, t (recovered pose)
  → Pinhole: same K (shared intrinsics)

Code: visualize()

rr.init("sfm_pipeline", spawn=True)

pts    = reconstruction.points
colors = (reconstruction.colors*255).astype('uint8')
rr.log("world/points",
    rr.Points3D(positions=pts, colors=colors, radii=0.01))

for i, cam in enumerate(reconstruction.cameras):
    rr.log(f"world/camera/{i}", rr.Transform3D(
        translation=cam.pose_matrix[:3, 3],
        mat3x3=cam.pose_matrix[:3, :3],
    ))
    rr.log(f"world/camera/{i}/image", rr.Pinhole(
        focal_length=[cam.K[0,0], cam.K[1,1]],
        principal_point=[cam.K[0,2], cam.K[1,2]],
        width=cam.width, height=cam.height,
    ))
Running the pipeline From the src/ directory:

python main.py uses the first two images in data/images/
python main.py --no-viz skips Rerun and only evaluates
python main.py --images a.jpg b.jpg specifies custom images

Evaluation plots are written to eval_output/. Rerun spawns a viewer window automatically.

What Worked, What Didn't

The RANSAC refit was the single biggest improvement. Without the final eight_point(inliers) call, error was ~1.2 px. With it: 0.50 px. Using 8 samples gives a valid F; using 900 inliers gives a good F.

Depth distribution has a gap. A visible trough between ~3.6 and 4.1 scene units reflects the architectural gap between the fountain wall and far background, a real scene feature, not an artefact.

The p95 error is higher than the mean. Both cameras show p95 around 1.1–1.2 px despite a 0.50 px mean. A small number of far-away or weakly-textured points reconstruct poorly, creating a heavy right tail.

Rerun has a learning curve for camera conventions. The Transform3D API expects the camera-to-world transform, but our R, t describe world-to-camera. Using cam.pose_matrix (camera pose in world coordinates) directly avoids an inversion bug that took a while to track down.

Series complete

That's the full casualSfM pipeline: SIFT matching → RANSAC F estimation → pose recovery → DLT triangulation → reprojection evaluation → Rerun visualisation. 701 coloured 3D points from two photographs, sub-pixel reprojection error, built entirely from NumPy and OpenCV primitives.
