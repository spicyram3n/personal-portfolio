The Fundamental Matrix
Normalised 8-Point Algorithm & RANSAC

Given ~1,500 noisy pixel matches, find the camera geometry. This chapter covers the epipolar constraint, Hartley normalisation, SVD-based 8-point algorithm, and adaptive RANSAC, each formula shown alongside the code that implements it. Source: geometry.py.

The Epipolar Constraint

A 3D point P visible in both images projects to p₁ in image 1 and p₂ in image 2. Even though we don't know where P is in 3D, we know that p₂ must lie on a specific line in image 2, the epipolar line of p₁. The Fundamental Matrix F encodes this constraint for all possible points.

Math

For any true match (p₁, p₂):

  p₂ᵀ F p₁ = 0

where pᵢ = (xᵢ, yᵢ, 1)ᵀ  (homogeneous)

F is 3×3, rank 2, 7 DOF
(9 entries − 1 scale − 1 rank constraint)

  F p₁   → epipolar line in image 2
  Fᵀ p₂  → epipolar line in image 1

What this enables

For an inlier match (p₁, p₂):
  p₂ᵀ F p₁  ≈  0  (small residual)

For an outlier (wrong match):
  p₂ᵀ F p₁  ≫  0  (large residual)

→ F is the inlier/outlier separator.
  RANSAC uses this to clean up the
  noisy match list from Ch 01.

The 8-Point Algorithm

Expanding p₂ᵀ F p₁ = 0 with p₁=(u,v,1) and p₂=(u',v',1) gives a linear equation in the 9 entries of F. Stack N such equations into a matrix system.

Math: one row of A per match

aᵢ = [u'u, u'v, u', v'u, v'v, v', u, v, 1]

A · vec(F) = 0,   A ∈ ℝᴺˣ⁹

Solution: vec(F) = last row of Vᵀ
where A = U Σ Vᵀ  (SVD)

Enforce rank-2 on recovered F:
  F = U diag(σ₁, σ₂, 0) Vᵀ
  (set smallest singular value to 0)

Code: eight_point()

A = np.column_stack([
    pts2_n[:,0]*pts1_n[:,0],  # u'u
    pts2_n[:,0]*pts1_n[:,1],  # u'v
    pts2_n[:,0],               # u'
    pts2_n[:,1]*pts1_n[:,0],  # v'u
    pts2_n[:,1]*pts1_n[:,1],  # v'v
    pts2_n[:,1],               # v'
    pts1_n[:,0], pts1_n[:,1], # u, v
    np.ones(len(pts1_n)),      # 1
])
_, _, Vt = np.linalg.svd(A)
F = Vt[-1].reshape(3, 3)     # ← solution

U, S, Vt = np.linalg.svd(F)
S[2] = 0.0                   # ← rank-2
F = U @ np.diag(S) @ Vt

Hartley Normalisation: Why It's Non-Negotiable

Pixel coordinates on our 3024×2016 images reach values up to ~3000. The A matrix contains products like u'u ≈ 9×10⁶. SVD on such a poorly conditioned matrix is dominated by floating-point rounding, so the recovered F is meaningless. Normalisation maps both point sets to zero mean, √2 average distance from origin.

Math

centroid:  c = (1/N) Σ pᵢ
scale:     s = √2 / mean(‖pᵢ − c‖)

Transform matrix:
T = | s   0  −s·cₓ |
    | 0   s  −s·cy |
    | 0   0    1   |

Normalised point:  p̃ = T p

After computing F in normalised coords:
  F_final = T₂ᵀ · F̃ · T₁

Code: normalize_points()

centroid = pts.mean(axis=0)
shifted  = pts - centroid
scale    = np.sqrt(2) / np.mean(
               np.linalg.norm(shifted, axis=1))

T = np.array([
    [scale, 0,    -scale*centroid[0]],
    [0,     scale, -scale*centroid[1]],
    [0,     0,      1               ],
])

pts_h    = np.hstack([pts, np.ones((len(pts),1))])
pts_norm = (T @ pts_h.T).T

# Denormalise after computing F:
F = T2.T @ F @ T1
return F / F[2, 2]   # fix scale (convention)
Challenge I hit Skipping normalisation gives mean reprojection error of ~18 px. With it: ~0.14 px on the best RANSAC sample, ~0.50 px after refitting on all inliers. Normalisation is not an optimisation. It is the difference between the algorithm working and not working.

Symmetric Epipolar Distance (SED)

To count inliers we need a geometric distance between a match and its epipolar line, in pixel units. The algebraic residual p₂ᵀFp₁ is not geometric. Its scale is arbitrary. SED sums the point-to-line distances in both images.

Math

Epipolar line in image 2:  l₂ = F p₁
Epipolar line in image 1:  l₁ = Fᵀ p₂

Algebraic residual:  r = |p₂ᵀ F p₁|

Point-to-line distance (pixels):
  d₂ = r / ‖(Fp₁)ₓᵧ‖₂
  d₁ = r / ‖(Fᵀp₂)ₓᵧ‖₂

SED = d₁ + d₂   (symmetric, in px)

Inlier criterion:  SED < 1.5 px

Code: symmetric_epipolar_distance()

p1h  = np.hstack([pts1, np.ones((len(pts1),1))])
p2h  = np.hstack([pts2, np.ones((len(pts2),1))])
Fp1  = (F   @ p1h.T).T   # lines in image 2
Ftp2 = (F.T @ p2h.T).T   # lines in image 1

# numerator: algebraic residual per match
num    = np.abs((p2h * Fp1).sum(axis=1))

# denominators: line direction norms (xy only)
denom1 = np.sqrt(Fp1 [:,0]**2 + Fp1 [:,1]**2)
denom2 = np.sqrt(Ftp2[:,0]**2 + Ftp2[:,1]**2)

return num/denom1 + num/denom2  # px

RANSAC: Adaptive Iteration Count

Even after the ratio test, ~30–40% of matches are outliers. Feeding them all into eight_point at once gives garbage. RANSAC draws minimal random samples (8 matches), estimates F, counts inliers via SED, and keeps the best.

Math

Let w = inlier fraction (estimated online).
Probability that 8 random samples are
all inliers: p_good = w⁸

Required iterations for confidence p:
  n = log(1−p) / log(1−w⁸)

As RANSAC finds better F → w↑ → n↓
→ terminate early.

Code: estimate_fundamental()

w_est   = 0.5          # initial guess: 50% inliers
n_iters = max(100, int(
    np.log(1 - 0.999) / np.log(1 - w_est**8)))
rng     = np.random.default_rng(0)  # reproducible

for iteration in range(n_iters):
    idx    = rng.choice(n, 8, replace=False)
    F_cand = eight_point(pts1[idx], pts2[idx])
    inlier = sed(F_cand, pts1, pts2) < threshold
    count  = inlier.sum()

    if count > best_count:
        best_count, best_mask, best_F = count, inlier, F_cand
        w_est   = count / n
        n_iters = min(n_iters,
            int(np.log(1-0.999) /
                np.log(1-max(w_est**8, 1e-10)))
            + iteration)

# Final refit on all inliers:
if best_mask.sum() >= 8:
    best_F = eight_point(pts1[best_mask], pts2[best_mask])
Key insight The final eight_point(pts1[best_mask], pts2[best_mask]) refit is crucial. The best RANSAC sample used only 8 points. Refitting on all ~900 inliers gives a much more stable F. This is what drives the final 0.50 px mean reprojection error.

Pitfalls

Normalise F after every eight_point call. The F / F[2,2] convention fixes scale across iterations. Without it, the SED threshold is effectively different each iteration, so inlier counts become incomparable.

Use a fixed random seed during development. np.random.default_rng(0) makes RANSAC deterministic, which you need when debugging why inlier counts fluctuate.

SED only uses the xy components of epipolar line vectors. The third component (the offset term) does not contribute to the geometric distance. Including it in the norm is a subtle but consequential bug. Always index [:, 0] and [:, 1] for the denominator.
