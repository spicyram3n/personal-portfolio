Pose Recovery & Triangulation
Essential Matrix to 3D Points

F gives us epipolar geometry in pixels. Multiplying in the camera intrinsics K gives the Essential Matrix E, the same geometry but in calibrated coordinates, from which we can recover the actual rotation R and translation t and triangulate 3D points. Source: geometry.py, camera.py, pipeline.py.

From F to the Essential Matrix

F relates pixel coordinates. E relates normalised image coordinates (pixels divided by the focal length and shifted by the principal point), which removes the camera's intrinsic parameters from the geometry. From E we can recover actual R and t.

Math

Normalised coordinate:
  x̃ = K⁻¹ · p   (p in homogeneous pixels)

Epipolar constraint in calibrated coords:
  x̃₂ᵀ E x̃₁ = 0

Relationship between E and F:
  E = Kᵀ F K

E has the same 7 DOF as F, but its
SVD has a special structure:
  E = U diag(1, 1, 0) Vᵀ
  (both non-zero singular values equal)

Code: recover_pose()

def recover_pose(F, pts1, pts2, K):
    E = K.T @ F @ K

    # cv.recoverPose handles decomposition
    # and cheirality check internally
    _, R, t, mask = cv.recoverPose(
        E, pts1, pts2, K
    )

    valid    = mask.ravel().astype(bool)
    pts1_in  = pts1[valid]
    pts2_in  = pts2[valid]
Key insight K here is the camera intrinsic matrix, not arbitrary but calibrated from a physical camera. For our fountain sequence: K = [[2759.48, 0, 1520.69], [0, 2764.16, 1006.81], [0, 0, 1]]. The large focal length (~2760 px) means a narrow field of view. If K is wrong, every downstream result is wrong.

Decomposing E into (R, t)

The SVD of E yields four candidate (R, t) pairs, four camera positions that are all geometrically consistent with the epipolar constraint. The correct one is selected by the cheirality condition: the triangulated 3D points must lie in front of both cameras.

Math: four solutions from SVD(E)

E = U Σ Vᵀ,  Σ = diag(σ, σ, 0)

W = | 0 −1  0 |
    | 1  0  0 |
    | 0  0  1 |

Four (R, t) candidates:
  (U W  Vᵀ, +U[:,2])
  (U W  Vᵀ, −U[:,2])
  (U Wᵀ Vᵀ, +U[:,2])
  (U Wᵀ Vᵀ, −U[:,2])

Cheirality: for each candidate (R, t),
  triangulate → keep the one where
  X[:,2] > 0 in both camera frames.

Code

# cv.recoverPose does SVD + cheirality:
#   - Computes all 4 (R, t) candidates
#   - Triangulates a subset of matches
#   - Picks (R, t) where most points
#     have positive depth in both cameras
#   - Returns mask of positive-depth inliers

_, R, t, mask = cv.recoverPose(E, pts1, pts2, K)

# R: 3×3 rotation matrix (cam1→cam2)
# t: 3×1 unit translation vector
#    (direction only, scale unknown)
# mask: 1 where cheirality passed

Triangulation: DLT Method

With R and t we know both camera positions. For each inlier match (p₁, p₂) we compute the 3D point X that best explains both observations using the Direct Linear Transform (DLT).

Math: projection model

Camera 1 at world origin:
  P₁ = K [I | 0]

Camera 2 at (R, t):
  P₂ = K [R | t]

For a 3D point X (homogeneous 4-vector):
  λ₁ p₁ = P₁ X
  λ₂ p₂ = P₂ X

Cross-multiplying to eliminate λ:
  p₁ × (P₁ X) = 0  →  two rows of A
  p₂ × (P₂ X) = 0  →  two rows of A

AX = 0  →  solved by SVD (X = last row Vᵀ)

Convert homogeneous → Euclidean:
  X_eucl = X_h[:3] / X_h[3]

Code: triangulation in recover_pose()

P1 = K @ np.hstack([np.eye(3),
                    np.zeros((3,1))])
P2 = K @ np.hstack([R, t])

# DLT triangulation (OpenCV)
X_h = cv.triangulatePoints(
    P1, P2,
    pts1_in.T,   # shape (2, N)
    pts2_in.T    # shape (2, N)
)
# X_h: 4×N homogeneous 3D points

X = (X_h[:3] / X_h[3]).T
# X: N×3 Euclidean 3D points

Cheirality Depth Filter

cv.recoverPose runs a cheirality check internally, but we apply one more pass: keep only points with strictly positive z-coordinate in the world frame. Negative depths mean the point reconstructed behind camera 1, a numerical artifact.

Math

Camera 1 is at world origin.
World z-axis = optical axis of camera 1.

Cheirality condition:
  X[2] > 0   (point is in front)

Also check in camera 2's frame:
  (R X + t)[2] > 0

cv.recoverPose enforces both.
Our additional filter enforces:
  X[:, 2] > 0  (world frame)

Code: pipeline.py run()

X, R, t, pts1_in, pts2_in = recover_pose(
    F, pts1, pts2, self.K
)

# Additional positive-depth filter:
ok = X[:, 2] > 0
X       = X[ok]
pts1_in = pts1_in[ok]
pts2_in = pts2_in[ok]

# On the fountain pair:
#   after recover_pose: ~900 points
#   after depth filter: 701 points

The Camera Model: camera.py

Each camera is stored as a dataclass with K, R, t and an image path. The key method is project, which re-projects 3D points back into the image, used in Ch 04 to compute reprojection error.

Math: perspective projection

Given world point X (3D):
  X_cam = R X + t       (world → camera)

Project via pinhole model:
  u = fx · (X_cam[0] / X_cam[2]) + cx
  v = fy · (X_cam[1] / X_cam[2]) + cy

Or compactly:
  λ p = K [R | t] X_world

Rodrigues:  R → rvec (axis-angle, 3-vector)
  used by cv.projectPoints internally.

Code: Camera.project()

@dataclass
class Camera:
    K: np.ndarray   # 3×3 intrinsics
    R: np.ndarray   # 3×3 rotation
    t: np.ndarray   # 3×1 translation
    image_path: str

    def project(self, pts3d):
        rvec, _ = cv.Rodrigues(self.R)
        proj, _ = cv.projectPoints(
            pts3d.astype(np.float64),
            rvec,
            self.t.astype(np.float64),
            self.K,
            None,   # no distortion (calibrated)
        )
        return proj.reshape(-1, 2)

The Full Pipeline: pipeline.py

SfMPipeline.run() is the orchestrator. It calls all three prior stages in sequence and packages the result into a Reconstruction dataclass. 

@dataclass
class Reconstruction:
    points:  np.ndarray        # (N, 3) float64, 3D point positions
    colors:  np.ndarray        # (N, 3) float32, RGB in [0, 1]
    cameras: list[Camera]      # two cameras: cam1 at origin, cam2 at (R, t)
    pts2d:   list[np.ndarray]  # [pts1_in, pts2_in], matched 2D observations


def run(self, img1_path, img2_path) -> Reconstruction:
    # Stage 1: feature matching (Ch 01)
    pts1, pts2 = self.matcher(img1_path, img2_path)

    # Stage 2: fundamental matrix + RANSAC (Ch 02)
    F, mask = estimate_fundamental(pts1, pts2, self.ransac_threshold, self.confidence)
    pts1, pts2 = pts1[mask], pts2[mask]

    # Stage 3: pose recovery + triangulation (this chapter)
    X, R, t, pts1_in, pts2_in = recover_pose(F, pts1, pts2, self.K)
    ok = X[:, 2] > 0
    X, pts1_in, pts2_in = X[ok], pts1_in[ok], pts2_in[ok]

    colors  = self._sample_colors(img2_path, pts2_in)
    cameras = [
        Camera.from_image(img1_path, self.K),
        Camera(self.K, R, t, img2_path, ...),
    ]
    return Reconstruction(X, colors, cameras, [pts1_in, pts2_in])

Colour Sampling

Each triangulated point gets the RGB colour of its 2D observation in image 2. The implementation rounds float pixel coordinates to integers and clips to image bounds.

Math

For each inlier point with image-2
observation (u, v):

  color = I₂[round(v), round(u)] / 255

  (clamp u to [0, W−1], v to [0, H−1])

Result: N×3 float32 array in [0, 1]

Code

img = cv.cvtColor(
    cv.imread(image_path), cv.COLOR_BGR2RGB)
h, w = img.shape[:2]

xs = np.clip(pts2d[:,0].astype(int), 0, w-1)
ys = np.clip(pts2d[:,1].astype(int), 0, h-1)

colors = img[ys, xs].astype(np.float32) / 255.0
# shape: (N, 3), dtype: float32

Scale Ambiguity

cv.recoverPose returns a unit translation vector, the direction of camera motion but not its magnitude. This is fundamental to monocular SfM: the same image pair can be explained by a camera moving 10 cm with a small scene, or 10 m with a proportionally larger scene. There is no way to determine absolute scale without external information. 

# t is a unit vector: ‖t‖₂ = 1.0
# The 3D point depths are in units of ‖t‖
# (relative to the baseline, not metric)

# To get metric scale you would need:
#   - A known object of known size in the scene
#   - Stereo camera with known baseline
#   - IMU integration

Pitfalls

Cheirality drops more points than expected. Far-away points are reconstructed near the epipole of the second camera, where a small angular error in F flips the depth sign. This is expected behaviour. Those points have poor triangulation geometry (small baseline relative to depth).

t is a direction, not a displacement. Using ‖t‖ as a scale factor in downstream calculations silently produces wrong absolute numbers.

det(R) must be +1. cv.recoverPose guarantees this, but if you implement your own decomposition, a −1 determinant gives a reflection, all points end up behind the camera.
